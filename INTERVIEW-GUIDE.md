# Technical Recruiter Interview Guide

This guide helps recruiters and hiring managers validate the portfolio in interviews.

## Security Architecture

| Question | What a strong answer should show |
| --- | --- |
| How do you apply Security by Design in a new project? | Risk classification, requirements, threat modeling, controls and evidence |
| How do you communicate architecture risk to executives? | Business impact, options, trade-offs and decision records |
| How do you prioritize security controls? | Risk, data sensitivity, exposure, regulation and operational feasibility |

## Cloud Security

| Question | What a strong answer should show |
| --- | --- |
| What should a secure Azure landing zone include? | IAM, network segmentation, logging, policy, monitoring and governance |
| How do you reduce excessive privileges in cloud? | RBAC, PIM, access reviews, least privilege and conditional access |

## DevSecOps and AppSec

| Question | What a strong answer should show |
| --- | --- |
| How do you add security to CI/CD without blocking delivery? | Quality gates, risk thresholds, automation and exception workflow |
| What do you check in an API security review? | Authentication, authorization, input validation, logging and rate limits |

## GRC and Risk

| Question | What a strong answer should show |
| --- | --- |
| How do you transform technical findings into GRC evidence? | Control mapping, owners, status, risk level and audit-ready artifacts |
| What makes a risk dashboard useful? | Clear metrics, prioritization, owners and executive decision support |

## AI Security and Governance

| Question | What a strong answer should show |
| --- | --- |
| How would you govern an enterprise AI use case? | Inventory, risk classification, impact assessment, controls and monitoring |
| How do you secure a RAG architecture? | Source governance, identity-aware retrieval, data classification, logging and response validation |
| What should be in a model card? | Purpose, intended use, data, limitations, evaluation, risks and monitoring |

## Automation

| Question | What a strong answer should show |
| --- | --- |
| What should be automated in security operations? | Evidence collection, reporting, access review support and repetitive checks |
| How do you keep automation safe? | Least privilege, logging, validation, dry runs and change control |

## Recruiter Scoring Rubric

| Signal | Weak | Strong |
| --- | --- | --- |
| Business context | Talks only about tools | Explains risk, impact and stakeholders |
| Architecture | Lists components | Explains trade-offs and controls |
| Governance | Mentions compliance generically | Maps controls to evidence and owners |
| Communication | Uses only jargon | Explains clearly for technical and executive audiences |
| Security maturity | Focuses on isolated fixes | Builds repeatable processes and decision models |
